# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.1.x   | ✅ Latest          |
| 1.0.x   | ⚠️ Limited Support |

## Reporting a Vulnerability

If you discover a security vulnerability within KOBEDORK, please follow these steps:

1. **Do not report security vulnerabilities through public GitHub issues**
2. Instead, send an email to the development team at cahyod@yahoo.co.id
3. Include the following information in your report:
   - A clear description of the vulnerability
   - Steps to reproduce the issue
   - Potential impact of the vulnerability
   - Any proof-of-concept code (if applicable)

### Response Timeline

- **Acknowledgment**: Within 48 hours of your report
- **Assessment**: Within 1 week of your report
- **Update**: Weekly updates until resolution if applicable
- **Resolution**: Timeline varies based on severity and complexity

## Security Scope

This security policy applies to the KOBEDORK application itself and its associated components. Please note:

- This application is a client-side tool that generates search queries
- It does not collect user data or store information on external servers
- All data is stored locally in the user's browser

## Security Features

KOBEDORK includes the following security considerations:

- Client-side only operation (no data sent to external servers)
- No user authentication required
- Local storage used for favorites and history
- No backend to potentially compromise
- No user data collection

## Responsible Disclosure

We appreciate responsible disclosure of security vulnerabilities. When reporting, please:

- Provide sufficient details to reproduce the issue
- Allow reasonable time for a response before public disclosure
- Work with us to understand and resolve the issue
- Respect the privacy of other users
- Follow ethical guidelines for security research

## Additional Security Information

- This tool is intended for legitimate security research and bug bounty hunting
- Always ensure you have proper authorization before testing any systems
- Comply with local laws and regulations when using this tool
- Use responsibly and ethically
